SAGA PATTERN - Complete Explanation
1. What is Saga Pattern?
Saga is a pattern to manage distributed transactions across multiple services where traditional ACID transactions don't work (because each service has its own database).
Two types:
- Choreography: Services publish/subscribe events to coordinate (no central orchestrator)
- Orchestration: A central orchestrator (Saga) tells each service what to do
2. Why Saga?
In microservices, you CAN'T do this:
BEGIN TRANSACTION
  INSERT INTO Payments...
  UPDATE Billing...
  UPDATE Card...
COMMIT
Each service has its OWN database. You need Saga to coordinate "all or nothing" across services.

True Saga compensation would look like:
1. PaymentService: Process payment → SUCCESS
2. BillingService: Mark bill paid → SUCCESS  
3. CardService: Deduct balance → FAIL!
4. Saga compensates:
   - BillingService: Un-mark bill paid
   - PaymentService: Refund payment
You have NONE of this compensation logic.

6. How to FIX It (The Right Saga Pattern)
You need a Saga Orchestrator that coordinates ALL services:
┌─────────────────────────────────────────────────────────────────┐
│                    SAGA ORCHESTRATOR                             │
│                                                                  │
│  1. ProcessPayment          → PaymentService                     │
│  2. MarkBillPaid            → BillingService                    │
│  3. DeductBalance           → CardService                       │
│  4. IF CardService FAILS →                                          │
│       CompensateBilling    → BillingService (undo step 2)       │
│       RefundPayment         → PaymentService (undo step 1)       │
└─────────────────────────────────────────────────────────────────┘
Implementation approach:
// Option 1: Orchestration-based Saga (recommended for your case)
public class PaymentOrchestrationSaga : MassTransitStateMachine<SagaState>
{
    // States: Started, PaymentProcessed, BillUpdated, CardDeducted, Completed, Compensating, Compensated
    
    public Event<IStartPaymentProcess> StartProcess { get; set; }
    
    When(StartProcess)
        // Step 1: Process payment
        .ForEachAsync(async ctx => 
        {
            await paymentService.Process(ctx.Saga.PaymentId);
            ctx.Saga.PaymentProcessedAt = DateTime.UtcNow;
        })
        .TransitionTo(PaymentProcessed)
        .Then(x => sagaRepo.Save(x.Saga))
        // Step 2: Update bill
        .ThenAsync(async ctx =>
        {
            await billingService.MarkPaid(ctx.Saga.BillId);
        })
        .TransitionTo(BillUpdated)
        // Step 3: Deduct card
        .ThenAsync(async ctx =>
        {
            await cardService.Deduct(ctx.Saga.CardId, ctx.Saga.Amount);
        })
        .TransitionTo(CardDeducted)
        .TransitionTo(Completed);
    
    // COMPENSATION: If ANY step fails
    DuringAny(When(IStepFailed)
        .TransitionTo(Compensating)
        .ForEachAsync(async ctx =>
        {
            // Compensate in REVERSE order!
            if (ctx.Saga.BillUpdated)
                await billingService.UnMarkPaid(ctx.Saga.BillId);
            if (ctx.Saga.PaymentProcessed)
                await paymentService.Refund(ctx.Saga.PaymentId);
        })
        .TransitionTo(Compensated));
}
