# IDENTITY SERVICE - COMPLETE LEARNING GUIDE
**Port: 5001 | Database: credvault_identity**

---

## What This Service Does

The Identity Service handles **all authentication and user management**. It's the gatekeeper for the entire system.

**Capabilities:**
- User registration with email/password
- Login with JWT token generation
- Google OAuth authentication
- Email verification via OTP
- Password reset via OTP
- Profile management
- Admin user management (status, role updates)

```csharp
    [MaxLength(16)]

    public DateTime? PasswordResetOtpExpiresAtUtc { get; set; }
    
    public UserStatus Status { get; set; }    // Account status
    public UserRole Role { get; set; }        // admin or user
    
    public DateTime CreatedAtUtc { get; set; }
    public DateTime UpdatedAtUtc { get; set; }
```

### 2. UserStatus.cs (Enum)

**What it is:** Tracks the lifecycle state of a user account.

```csharp
public enum UserStatus
{
    PendingVerification = 1,   // Just registered, needs email verification
    Active = 2,               // Verified, can use system
    Suspended = 3,            // Temporarily disabled
    Blocked = 4,              // Permanently blocked
    Deleted = 5               // Soft deleted
}
```

**Why it matters:** Users are NEVER hard-deleted. Deleted just changes status.

#### RegisterCommandHandler.cs

**Read this file to understand:**
1. How MediatR handlers work
2. Validation flow
3. Password hashing with BCrypt
4. OTP generation
5. Event publishing
6. Error handling

**Step-by-step flow:**

```csharp
public class RegisterCommandHandler : IRequestHandler<RegisterCommand, ApiResponse<AuthResponseDto>>
{
    // Dependencies injected via constructor
    private readonly IUserRepository _userRepository;
    private readonly IPublishEndpoint _publishEndpoint;
    private readonly ILogger<RegisterCommandHandler> _logger;
    
    public RegisterCommandHandler(
        IUserRepository userRepository,
        IPublishEndpoint publishEndpoint,
        ILogger<RegisterCommandHandler> logger)
    {
        _userRepository = userRepository;
        _publishEndpoint = publishEndpoint;
        _logger = logger;
    }
    
    public async Task<ApiResponse<AuthResponseDto>> Handle(
        RegisterCommand request, 
        CancellationToken cancellationToken)
    {
        // STEP 1: Check if email already exists
        if (await _userRepository.EmailExistsAsync(request.Email))
        {
            return ApiResponse<AuthResponseDto>.Fail("Email already registered");
        }
        
        // STEP 2: Generate OTP for email verification
        var otp = IdentityHelpers.GenerateOtp(); // 6-digit random number
        
        // STEP 3: Create user entity
        var user = new IdentityUser
        {
            Id = Guid.NewGuid(),
            Email = request.Email,
            FullName = request.FullName,
            PasswordHash = BCrypt.Net.BCrypt.HashPassword(request.Password), // Hash password
            IsEmailVerified = false,
            EmailVerificationOtp = otp,
            EmailVerificationOtpExpiresAtUtc = DateTime.UtcNow.AddMinutes(15),
            Status = UserStatus.PendingVerification,
            Role = UserRole.User,
            CreatedAtUtc = DateTime.UtcNow,
            UpdatedAtUtc = DateTime.UtcNow
        };
        
        // STEP 4: Save to database
        await _userRepository.CreateAsync(user);
        
        // STEP 5: Publish event for notification service
        await _publishEndpoint.Publish(new IUserRegistered
        {
            UserId = user.Id,
            Email = user.Email,
            FullName = user.FullName,
            Otp = otp  // In real app, you'd want to mask this or send via separate secure channel
        });
        
        _logger.LogInformation("User registered: {Email}", user.Email);
        
        // STEP 6: Return success response
        return ApiResponse<AuthResponseDto>.Success(
            new AuthResponseDto 
            { 
                UserId = user.Id,
                Message = "Registration successful. Please verify your email."
            },
            "Registration successful"
        );
    }
}
```

### 3. LoginCommand & Handler

**Location:** `IdentityService.Application/Commands/Login/`

**What it does:** Authenticates user and returns JWT token.

**Key concepts to learn:**
1. BCrypt password verification
2. JWT token structure
3. Claim-based identity
4. Token expiration

```csharp
public class LoginCommandHandler : IRequestHandler<LoginCommand, ApiResponse<AuthResponseDto>>
{
    public async Task<ApiResponse<AuthResponseDto>> Handle(LoginCommand request, CancellationToken cancellationToken)
    {
        // 1. Get user by email
        var user = await _userRepository.GetByEmailAsync(request.Email);
        
        // 2. Check if user exists
        if (user == null)
        {
            return ApiResponse<AuthResponseDto>.Fail("Invalid credentials");
        }
        
        // 3. Check password
        if (!BCrypt.Net.BCrypt.Verify(request.Password, user.PasswordHash))
        {
            return ApiResponse<AuthResponseDto>.Fail("Invalid credentials");
        }
        
        // 4. Check if email verified
        if (!user.IsEmailVerified)
        {
            return ApiResponse<AuthResponseDto>.Fail("Please verify your email first");
        }
        
        // 5. Check account status
        if (user.Status != UserStatus.Active)
        {
            return ApiResponse<AuthResponseDto>.Fail($"Account is {user.Status}");
        }
        
        // 6. Generate JWT
        var token = IdentityHelpers.GenerateJwtToken(user);
        
        return ApiResponse<AuthResponseDto>.Success(new AuthResponseDto
        {
            Token = token,
            ExpiresAt = DateTime.UtcNow.AddHours(24), // From JwtOptions
            UserId = user.Id,
            Email = user.Email,
            FullName = user.FullName,
            Role = user.Role.ToString()
        });
    }
}
```

### 4. IdentityHelpers.cs

**Location:** `IdentityService.Application/Helpers/IdentityHelpers.cs`

**Read this file to understand:**
- JWT token generation
- OTP generation
- Entity to DTO mapping

**Key methods:**

```csharp
public static class IdentityHelpers
{
    // JWT Generation
    public static string GenerateJwtToken(IdentityUser user)
    {
        // 1. Create claims (user info embedded in token)
        var claims = new[]
        {
            new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()),
            new Claim(ClaimTypes.Email, user.Email),
            new Claim(ClaimTypes.Name, user.FullName),
            new Claim(ClaimTypes.Role, user.Role.ToString())
        };
        
        // 2. Create signing credentials (uses secret key from JwtOptions)
        var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(jwtOptions.SecretKey));
        var creds = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);
        
        // 3. Create token descriptor
        var token = new JwtSecurityToken(
            issuer: jwtOptions.Issuer,
            audience: jwtOptions.Audience,
            claims: claims,
            expires: DateTime.UtcNow.AddMinutes(jwtOptions.AccessTokenMinutes),
            signingCredentials: creds
        );
        
        // 4. Return token string
        return new JwtSecurityTokenHandler().WriteToken(token);
    }
    
    // OTP Generation
    public static string GenerateOtp()
    {
        // 6-digit random number
        var random = new Random();
        return random.Next(100000, 999999).ToString();
    }
    
    // Entity to DTO mapping
    public static UserDto ToDto(IdentityUser user)
    {
        return new UserDto
        {
            Id = user.Id,
            Email = user.Email,
            FullName = user.FullName,
            IsEmailVerified = user.IsEmailVerified,
            Status = user.Status.ToString(),
            Role = user.Role.ToString(),
            CreatedAtUtc = user.CreatedAtUtc
        };
    }
}
```

## Infrastructure Layer - Understanding Each File

### 1. IdentityDbContext.cs

**Location:** `IdentityService.Infrastructure/Persistence/Sql/IdentityDbContext.cs`

**What it is:** Entity Framework Core database context.

**Read it to understand:**
- Database table configuration
- Column mappings
- Enum conversions (stored as int in DB)

```csharp
public class IdentityDbContext : DbContext
{
    public DbSet<IdentityUser> Users { get; set; }
    
    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        
        modelBuilder.Entity<IdentityUser>(entity =>
        {
            // Table name
            entity.ToTable("identity_users");
            
            // Primary key
            entity.HasKey(e => e.Id);
            
            // Column configurations
            entity.Property(e => e.Email)
                .IsRequired()
                .HasMaxLength(256);
            
            entity.HasIndex(e => e.Email)
                .IsUnique(); // Email must be unique
            
            // Enum stored as int in database
            entity.Property(e => e.Status)
                .HasConversion<int>();
            
            entity.Property(e => e.Role)
                .HasConversion<int>();
            
            // Audit fields
            entity.Property(e => e.CreatedAtUtc)
                .IsRequired();
            
            entity.Property(e => e.UpdatedAtUtc)
                .IsRequired();
        });
    }
}
```

### 2. SqlUserRepository.cs

**Location:** `IdentityService.Infrastructure/Persistence/Sql/Repositories/SqlUserRepository.cs`

**Read this to understand:**
- Repository implementation pattern
- EF Core queries
- Async/await patterns

```csharp
public class SqlUserRepository : IUserRepository
{
    private readonly IdentityDbContext _context;
    
    public SqlUserRepository(IdentityDbContext context)
    {
        _context = context;
    }
    
    public async Task<IdentityUser?> GetByIdAsync(Guid id)
    {
        return await _context.Users
            .AsNoTracking() // Performance: don't track for read-only
            .FirstOrDefaultAsync(u => u.Id == id);
    }
    
    public async Task<IdentityUser?> GetByEmailAsync(string email)
    {
        return await _context.Users
            .AsNoTracking()
            .FirstOrDefaultAsync(u => u.Email.ToLower() == email.ToLower());
    }
    
    public async Task<IdentityUser> CreateAsync(IdentityUser user)
    {
        _context.Users.Add(user);
        await _context.SaveChangesAsync();
        return user;
    }
    
    public async Task<IdentityUser> UpdateAsync(IdentityUser user)
    {
        user.UpdatedAtUtc = DateTime.UtcNow; // Always update timestamp
        _context.Users.Update(user);
        await _context.SaveChangesAsync();
        return user;
    }
    
    public async Task<List<IdentityUser>> GetAllAsync(int page, int pageSize)
    {
        return await _context.Users
            .AsNoTracking()
            .OrderByDescending(u => u.CreatedAtUtc)
            .Skip((page - 1) * pageSize)
            .Take(pageSize)
            .ToListAsync();
    }
    
    public async Task<Dictionary<UserStatus, int>> GetStatusCountsAsync()
    {
        return await _context.Users
            .GroupBy(u => u.Status)
            .Select(g => new { Status = g.Key, Count = g.Count() })
            .ToDictionaryAsync(x => x.Status, x => x.Count);
    }
}
```

```csharp
    {

    }

    {
        var result = await _mediator.Send(command);

    }
    
    [HttpPost("login")]
    public async Task<IActionResult> Login([FromBody] LoginCommand command)
    {
        var result = await _mediator.Send(command);
        
        if (!result.Success)
            return Unauthorized(result); // 401
        
        return Ok(result);
    }
    
    [HttpPost("verify-email")]
    public async Task<IActionResult> VerifyEmail([FromBody] VerifyEmailOtpCommand command)
    {
        var result = await _mediator.Send(command);
        return Ok(result);
    }
    
    [HttpPost("forgot-password")]
    public async Task<IActionResult> ForgotPassword([FromBody] ForgotPasswordCommand command)
    {
        var result = await _mediator.Send(command);
        return Ok(result); // Always return 200 (don't reveal if email exists)
    }
    
    [HttpPost("reset-password")]
    public async Task<IActionResult> ResetPassword([FromBody] ResetPasswordCommand command)
    {
        var result = await _mediator.Send(command);
        return Ok(result);
    }
```

### 3. Program.cs (Dependency Injection)

**Location:** `IdentityService.API/Program.cs`

```csharp
var builder = WebApplication.CreateBuilder(args);

// 1. Add Controllers
builder.Services.AddControllers();

// 2. Add JWT Authentication (from Shared Contracts)
builder.Services.AddJwtAuthentication(jwtOptions);

// 3. Add Database Context
builder.Services.AddDbContext<IdentityDbContext>(options =>
    options.UseSqlServer(connectionString));

// 6. Add MassTransit (for event publishing)
builder.Services.AddMassTransit(x =>
{
    x.AddConsumer<UserRegisteredConsumer>(); // If any
    x.UsingRabbitMq(...);
});

// 7. Add Swagger (API documentation)
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

// Middleware pipeline
if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseExceptionHandling(); // From Shared Contracts
app.UseAuthentication();    // JWT
app.UseAuthorization();     // Role-based
```

## Exercise: Trace a Registration

1. Client sends `POST /api/v1/identity/auth/register`
2. Which controller method handles this? → `AuthController.Register()`
3. What does it do? → Sends `RegisterCommand` to MediatR
4. Who handles the command? → `RegisterCommandHandler.Handle()`
5. What does the handler do?
   - Check if email exists? (`_userRepository.EmailExistsAsync()`)
   - Generate OTP? (`IdentityHelpers.GenerateOtp()`)
   - Hash password? (`BCrypt.HashPassword()`)
   - Create user? (`_userRepository.CreateAsync()`)
   - Publish event? (`_publishEndpoint.Publish()`)
6. What's the response? → `ApiResponse<AuthResponseDto>`
