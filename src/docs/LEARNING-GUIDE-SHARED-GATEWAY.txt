# SHARED CONTRACTS & API GATEWAY - LEARNING GUIDE

---

## PART 1: SHARED CONTRACTS LIBRARY

**Location:** `/server/shared.contracts/Shared.Contracts/`

This is the **glue** that holds all microservices together. Every service references this library.

---

### Why Shared Contracts?

In microservices, services need to communicate. Without shared contracts:

```
Problem:
Service A sends: { "user_id": "abc", "amount": 100 }
Service B expects: { "userId": "abc", "Amount": 100 }

Result: Mismatch, bugs, failures
```

With shared contracts:

```
Solution:
Shared.Contracts defines: IUserRegistered { Guid UserId, decimal Amount }
All services use the same definition

Result: Type safety, compile-time checking
```

## Understanding Each File

### 1. ApiResponse.cs (Standard Response)

**Location:** `Shared.Contracts/Models/ApiResponse.cs`

**What it is:** Standard wrapper for all API responses.

```csharp
public class ApiResponse<T>
{
    public bool Success { get; set; }
    public string Message { get; set; }
    public T? Data { get; set; }
    public string? TraceId { get; set; }
    public List<string>? Errors { get; set; }
    
    // Factory methods
    public static ApiResponse<T> Success(T data, string message = "Success")
    {
        return new ApiResponse<T>
        {
            Success = true,
            Message = message,
            Data = data,
            TraceId = GetTraceId()
        };
    }
    
    public static ApiResponse<T> Fail(string message, List<string>? errors = null)
    {
        return new ApiResponse<T>
        {
            Success = false,
            Message = message,
            Errors = errors,
            TraceId = GetTraceId()
        };
    }
}

// Non-generic version for void operations
public class ApiResponse : ApiResponse<object>
{
    public static ApiResponse Success(string message = "Success")
    {
        return new ApiResponse { Success = true, Message = message };
    }
    
    public static ApiResponse Fail(string message)
    {
        return new ApiResponse { Success = false, Message = message };
    }
}
```

**Usage:**
```csharp
// Success response
return ApiResponse<CardDto>.Success(card, "Card created");

// Fail response
return ApiResponse<CardDto>.Fail("Card not found");

// Void success
return ApiResponse.Success("Operation completed");
```

### 2. BaseApiController.cs

**Location:** `Shared.Contracts/Controllers/BaseApiController.cs`

**What it is:** Base controller providing common functionality to all controllers.

```csharp
public abstract class BaseApiController : ControllerBase
{
    protected Guid GetUserId()
    {
        // Get user ID from JWT claims
        var userIdClaim = User.FindFirst(ClaimTypes.NameIdentifier);
        if (userIdClaim == null || !Guid.TryParse(userIdClaim.Value, out var userId))
        {
            throw new UnauthorizedAccessException("Invalid user token");
        }
        return userId;
    }
    
    protected string GetUserEmail()
    {
        var emailClaim = User.FindFirst(ClaimTypes.Email);
        return emailClaim?.Value ?? "";
    }
    
    protected string GetUserRole()
    {
        var roleClaim = User.FindFirst(ClaimTypes.Role);
        return roleClaim?.Value ?? "User";
    }
    
    protected bool IsAdmin()
    {
        return GetUserRole() == "Admin";
    }
    
    protected string? GetTraceId()
    {
        return HttpContext.TraceIdentifier;
    }
}
```

**Every service controller inherits from this:**
```csharp
public class CardsController : BaseApiController  // Not ControllerBase!
{
    [HttpGet]
    [Authorize]
    public async Task<IActionResult> GetCards()
    {
        var userId = GetUserId();  // Available from BaseApiController
        // ...
    }
}
```

### 3. ExceptionHandlingMiddleware.cs

**Location:** `Shared.Contracts/Middleware/ExceptionHandlingMiddleware.cs`

**What it is:** Global exception handler ensuring consistent error responses.

```csharp
public class ExceptionHandlingMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<ExceptionHandlingMiddleware> _logger;
    
    public async Task InvokeAsync(HttpContext context)
    {
        try
        {
            await _next(context);
        }
        catch (Exception ex)
        {
            await HandleExceptionAsync(context, ex);
        }
    }
    
    private async Task HandleExceptionAsync(HttpContext context, Exception exception)
    {
        _logger.LogError(exception, "Unhandled exception occurred");
        
        var (statusCode, message) = exception switch
        {
            UnauthorizedAccessException _ => (StatusCodes.Status401Unauthorized, "Unauthorized"),
            ArgumentException ex => (StatusCodes.Status400BadRequest, ex.Message),
            KeyNotFoundException _ => (StatusCodes.Status404NotFound, "Resource not found"),
            InvalidOperationException ex => (StatusCodes.Status400BadRequest, ex.Message),
            _ => (StatusCodes.Status500InternalServerError, "An error occurred")
        };
        
        context.Response.StatusCode = statusCode;
        context.Response.ContentType = "application/json";
        
        var response = new
        {
            success = false,
            message = message,
            traceId = context.TraceIdentifier
        };
        
        await context.Response.WriteAsync(JsonSerializer.Serialize(response));
    }
}
```

**Registration:**
```csharp
// In Program.cs
app.UseExceptionHandling();  // Extension method from Shared.Contracts
```

### 5. ServiceCollectionExtensions.cs

**Location:** `Shared.Contracts/Extensions/ServiceCollectionExtensions.cs`

**What it is:** Extension methods for common DI registration.

```csharp
public static class ServiceCollectionExtensions
{
    // Add JWT Authentication
    public static IServiceCollection AddJwtAuthentication(
        this IServiceCollection services,
        JwtOptions jwtOptions)
    {
        services.AddAuthentication(options =>
        {
            options.DefaultAuthenticateScheme = JwtBearerDefaults.AuthenticationScheme;
            options.DefaultChallengeScheme = JwtBearerDefaults.AuthenticationScheme;
        })
        .AddJwtBearer(options =>
        {
            options.TokenValidationParameters = new TokenValidationParameters
            {
                ValidateIssuer = true,
                ValidateAudience = true,
                ValidateLifetime = true,
                ValidateIssuerSigningKey = true,
                ValidIssuer = jwtOptions.Issuer,
                ValidAudience = jwtOptions.Audience,
                IssuerSigningKey = new SymmetricSecurityKey(
                    Encoding.UTF8.GetBytes(jwtOptions.SecretKey))
            };
        });
        
        return services;
    }
    
    // Add API Services (controllers, swagger, etc.)
    public static IServiceCollection AddApiServices(this IServiceCollection services)
    {
        services.AddControllers();
        services.AddEndpointsApiExplorer();
        services.AddSwaggerGen(c =>
        {
            c.SwaggerDoc("v1", new OpenApiInfo 
            { 
                Title = "CredVault API", 
                Version = "v1" 
            });
            
            // Add JWT auth to Swagger
            c.AddSecurityDefinition("Bearer", new OpenApiSecurityScheme
            {
                Description = "JWT Authorization header using the Bearer scheme",
                Name = "Authorization",
                In = ParameterLocation.Header,
                Type = SecuritySchemeType.ApiKey,
                Scheme = "Bearer"
            });
        });
        
        return services;
    }
}
```

**Usage in Program.cs:**
```csharp
builder.Services.AddJwtAuthentication(jwtOptions);
builder.Services.AddApiServices();
```

## PART 2: API GATEWAY

**Location:** `/server/Gateway.API/`

The API Gateway is the **single entry point** for all clients. It routes requests to appropriate microservices.

---

### Why API Gateway?

```
Without Gateway:
Client → Identity Service (5001)
Client → Card Service (5002)
Client → Billing Service (5003)
... (client needs to know all service URLs)

With Gateway:
Client → API Gateway (5006) → Routes to appropriate service
```

**Benefits:**
- Single entry point
- Authentication/authorization in one place
- Rate limiting
- Request routing
- Protocol translation

---

### Program.cs (Gateway)

```csharp
var builder = WebApplication.CreateBuilder(args);

// Add Ocelot
builder.Services.AddOcelot();

// Add JWT Authentication (same as services)
builder.Services.AddJwtAuthentication(jwtOptions);

// Add services for dependency injection
builder.Services.AddControllers();

// Add Swagger
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerForOcelot();

var app = builder.Build();

// Use Ocelot
app.UseSwaggerForOcelotUI();
app.UseOcelot().Wait();

app.Run();
```

### How Requests Flow Through Gateway

```
Client Request: POST /api/v1/payments/initiate

1. Request hits Gateway (port 5006)
2. Ocelot reads ocelot.json
3. Matches /api/v1/payments/{everything} route
4. Forwards to: http://localhost:5004/api/v1/payments/initiate
5. Payment Service processes request
6. Response returned to Gateway
7. Gateway returns response to Client
```

## Understanding Dependency Injection Setup

All services follow the same pattern in Program.cs:

```csharp
// 1. Add Controllers
builder.Services.AddControllers();

// 2. Add JWT Authentication (from Shared.Contracts)
builder.Services.AddJwtAuthentication(jwtOptions);

// 3. Add Database Context
builder.Services.AddDbContext<MyDbContext>(options =>
    options.UseSqlServer(connectionString));

// 4. Add Repository
builder.Services.AddScoped<IMyRepository, SqlMyRepository>();

// 5. Add MediatR (discovers all Commands/Queries)
builder.Services.AddMediatR(cfg =>
    cfg.RegisterServicesFromAssembly(typeof(CreateItemCommand).Assembly));

// 6. Add MassTransit (for RabbitMQ)
builder.Services.AddMassTransit(x =>
{
    // Add consumers
    x.AddConsumer<MyConsumer>();
    
    // Configure RabbitMQ
    x.UsingRabbitMq((context, cfg) =>
    {
        cfg.Host("localhost", "/", h => {
            h.Username("guest");
            h.Password("guest");
        });
        
        cfg.ReceiveEndpoint("my-queue", e =>
        {
            e.ConfigureConsumer<MyConsumer>(context);
        });
    });
});

// 7. Add Swagger
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

// 8. Add Background Services
builder.Services.AddHostedService<MyBackgroundService>();

// 9. Add HTTP Clients (for service-to-service calls)
builder.Services.AddHttpClient<ICardServiceClient, CardServiceClient>();

var app = builder.Build();

// Middleware Pipeline
if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseExceptionHandling();  // From Shared.Contracts
app.UseAuthentication();
app.UseAuthorization();

app.MapControllers();

app.Run();
```

## Service-to-Service Communication

### 1. Via Events (Asynchronous)

Used for fire-and-forget operations:

```csharp
// Publishing
await _publishEndpoint.Publish(new ICardAdded
{
    CardId = card.Id,
    UserId = card.UserId,
    Last4 = card.Last4
});

// Consuming (in another service)
public class CardAddedConsumer : IConsumer<ICardAdded>
{
    public async Task Consume(ConsumeContext<ICardAdded> context)
    {
        var message = context.Message;
        // Handle the event
    }
}
```

### 2. Via HTTP (Synchronous)

Used when you need immediate response:

```csharp
// Define HTTP client interface
public interface ICardServiceClient
{
    Task<CardDto?> GetCardAsync(Guid cardId);
    Task<bool> ValidateCardOwnershipAsync(Guid cardId, Guid userId);
}

// Implementation
public class CardServiceClient : ICardServiceClient
{
    private readonly HttpClient _httpClient;
    
    public CardServiceClient(HttpClient httpClient)
    {
        _httpClient = httpClient;
        _httpClient.BaseAddress = new Uri("http://localhost:5002");
    }
    
    public async Task<CardDto?> GetCardAsync(Guid cardId)
    {
        var response = await _httpClient.GetAsync($"/api/v1/cards/{cardId}");
        if (!response.IsSuccessStatusCode) return null;
        
        var content = await response.Content.ReadAsStringAsync();
        return JsonSerializer.Deserialize<ApiResponse<CardDto>>(content)?.Data;
    }
}

// Registration
builder.Services.AddHttpClient<ICardServiceClient, CardServiceClient>();

// Usage
public class MyHandler : IRequestHandler<MyCommand, ApiResponse<MyResult>>
{
    private readonly ICardServiceClient _cardService;
    
    public MyHandler(ICardServiceClient cardService)
    {
        _cardService = cardService;
    }
    
    public async Task<ApiResponse<MyResult>> Handle(MyCommand request, CancellationToken ct)
    {
        var card = await _cardService.GetCardAsync(request.CardId);
        // ...
    }
}
```
